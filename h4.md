# H4 - Pizza Fantasia

## Tiivistelmät

### 4.12.1 Size and Complexity of Some DSLs
- Konfiguraatiokielet (DSL) voivat kasvaa monimutkaisiksi ja sisältää paljon ominaisuuksia
- Ominaisuuksien määrä ei aina tarkoita parempaa käytettävyyttä
- Liian monimutkainen DSL tekee konfiguraatiosta vaikeasti ymmärrettävää
- Yksinkertaisempi kieli voi olla helpompi ylläpitää ja käyttää

**Oma huomio: Ansible on helppo, koska YAML on yksinkertainen – ei tarvitse opetella monimutkaista kieltä.**

### 4.12.2 Use of DSL Functions in Case Configuration
- Käytännössä käytetään vain pientä osaa DSL:n ominaisuuksista
- Yleisimmät asiat:
  - funktiot
  - ehtolauseet
  - perusrakenteet
- Suuri osa ominaisuuksista jää käyttämättä

**Oma huomio: Tämä näkyy myös meidän tehtävissä – käytetään vain basic taskeja.**

### 4.12.3.1 Dependencies Between Main Functions
- Konfiguraatiot koostuvat riippuvuuksista (dependencies)
- Esim:
  - paketti → tiedosto → palvelu
- Näiden järjestys on tärkeä
- Hyvä CM-työkalu hoitaa riippuvuudet automaattisesti

**Oma huomio: Tämä on suoraan Ansiblea: package → file → service.**

## a.) Räpylä

Tässä tehtävässä tavoitteena oli asentaa jokin demoni käsin ja varmistaa sen toiminta käytännössä. Valitsin asennettavaksi demoniksi PostgreSQL-tietokantapalvelimen. Valitsin tämän, koska se on yleisesti käytetty palvelinohjelmisto ja sopii hyvin demonien hallinnan harjoitteluun.
Aloitin päivittämällä pakettilistat, jotta asennettavat paketit ovat ajan tasalla. Tämän jälkeen asensin PostgreSQL:n apt-paketinhallinnan avulla.

<img width="1276" height="247" alt="image" src="https://github.com/user-attachments/assets/d47923b8-ce98-479f-a0c6-d82e083fd38d" /> **Katsoin päivitykset 'sudo apt update' komennolla ja päivitin 'sudo apt upgrade' -komennolla. Nyt päivitykset ovat ajantasalla ennen asennusta.**

<img width="1281" height="594" alt="image" src="https://github.com/user-attachments/assets/50b65673-2d1a-4890-be6b-69bbe72a85fa" /> **Asensin PostgreSQL-tietokantapalvelimen apt-paketinhallinnan avulla.**

Asennuksen jälkeen käynnistin palvelun ja tarkistin sen tilan varmistaakseni, että se toimii oikein. Palvelun tila oli aktiivinen (active running), mikä osoittaa että PostgreSQL on käynnissä.

<img width="1272" height="365" alt="image" src="https://github.com/user-attachments/assets/ffc8ca13-7b71-46ab-81e0-76ae68b2bfcd" /> **Käynnistin palvelimen 'sudo systemctl start postgresql' -komennolla. Tarkistin tämän jälkeen palvelimen tilan 'sudo systemctl status postgresql' -komennolla. Palvelin on nyt käynnissä (active running).**

Varmistin lisäksi, että palvelu käynnistyy automaattisesti järjestelmän käynnistyessä. Tämä on tärkeää palvelinkäytössä, jotta palvelu on aina käytettävissä ilman manuaalista käynnistystä.

<img width="1277" height="186" alt="image" src="https://github.com/user-attachments/assets/c0d88d2d-e42c-436c-89a3-958383d670ca" /> **Varmistin, että palvelu käynnistyy automaattisesti 'sudo systemctl enable postgresql' -komennolla.**

Lopuksi testasin palvelun toiminnan käytännössä kirjautumalla PostgreSQL:ään postgres-käyttäjänä. Onnistunut kirjautuminen osoitti, että palvelin toimii oikein eikä pelkkä asennus ole epäonnistunut.
