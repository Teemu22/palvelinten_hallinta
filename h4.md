# H4 - Pizza Fantasia

## Tiivistelmät

### 4.12.1 Size and Complexity of Some DSLs
- Konfiguraatiokielet (DSL) voivat kasvaa monimutkaisiksi ja sisältää paljon ominaisuuksia
- Ominaisuuksien määrä ei aina tarkoita parempaa käytettävyyttä
- Liian monimutkainen DSL tekee konfiguraatiosta vaikeasti ymmärrettävää
- Yksinkertaisempi kieli voi olla helpompi ylläpitää ja käyttää

**Oma huomio: Ansible on helppo, koska YAML on yksinkertainen – ei tarvitse opetella monimutkaista kieltä.**

### 4.12.2 Use of DSL Functions in Case Configuration
- Käytännössä käytetään vain pientä osaa DSL:n ominaisuuksista
- Yleisimmät asiat:
  - funktiot
  - ehtolauseet
  - perusrakenteet
- Suuri osa ominaisuuksista jää käyttämättä

**Oma huomio: Tämä näkyy myös meidän tehtävissä – käytetään vain basic taskeja.**

### 4.12.3.1 Dependencies Between Main Functions
- Konfiguraatiot koostuvat riippuvuuksista (dependencies)
- Esim:
  - paketti → tiedosto → palvelu
- Näiden järjestys on tärkeä
- Hyvä CM-työkalu hoitaa riippuvuudet automaattisesti

**Oma huomio: Tämä on suoraan Ansiblea: package → file → service.**
