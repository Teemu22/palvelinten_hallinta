# H5 - Gitar Hero

## Tiivistelmät

### What is Git?
- Git on versionhallintajärjestelmä, jonka avulla voidaan seurata tiedostoihin tehtyjä muutoksia ajan kuluessa
- Se mahdollistaa palaamisen aiempiin versioihin ja muutosten vertailun
- Git tallentaa projektista tilannekuvia (snapshot), ei pelkästään muutoksia
- Jokaisella käyttäjällä on oma paikallinen kopio projektista
- Muutokset voidaan jakaa muiden kanssa esimerkiksi GitHubin kautta

**Oma huomio: Git ei ole pelkkä tiedostojen siirtotyökalu, vaan sen pääidea on muutosten hallinta ja yhteistyön helpottaminen.**

### Gitin peruskomennot
- ```git add``` lisää muutetut tai uudet tiedostot versionhallintaan (staging area)
- ```git commit``` tallentaa muutokset ja luo uuden version projektista
- ```git push``` lähettää paikalliset muutokset etävarastoon (esim. GitHubiin)
- ```git pull``` hakee etävaraston muutokset paikalliseen projektiin
- ```git add``` valitsee, mitä muutoksia halutaan mukaan seuraavaan commitiin
- ```git commit``` vaatii viestin, jossa kuvataan mitä muutoksia on tehty
- ```git push``` tekee muutokset näkyväksi muille käyttäjille
- ```git pull``` varmistaa, että oma versio on ajan tasalla ennen uusien muutosten tekemistä

**Oma huomio: Näitä komentoja käytetään yleensä yhdessä järjestyksessä add → commit → push, ja pull ennen sitä, jotta vältetään ristiriidat.**

### a.) Online

Tässä tehtävässä luotiin uusi GitHub-repository ja yhdistettiin se paikalliseen koneeseen Gitin avulla. Tavoitteena oli harjoitella versionhallinnan perustoimintoja, kuten repositorion luontia, kloonausta, tiedostojen lisäämistä sekä muutosten viemistä GitHubiin.

Aluksi loin GitHubiin uuden repositoryn nimeltä "sunshine". Repositoryyn lisättiin heti luontivaiheessa README-tiedosto sekä GNU GPLv3 -lisenssi.

Repositoryn luomisen jälkeen se kloonattiin paikalliselle koneelle git clone -komennolla. Tämä mahdollistaa sen, että projektia voidaan muokata omalla koneella ja muutokset voidaan myöhemmin lähettää takaisin GitHubiin.

<img width="1279" height="324" alt="image" src="https://github.com/user-attachments/assets/de58a7bf-9604-4b84-ac0b-45cf1d15175f" /> **Kuvassa GitHub-repository kloonataan paikalliselle koneelle git clone -komennolla ja siirrytään projektikansioon.**

Kloonauksen jälkeen siirryin projektikansioon ja tarkistin sen sisällön. Kansiossa näkyivät GitHubissa luodut README.md- ja LICENSE-tiedostot, mikä vahvisti että kloonaus onnistui oikein.

<img width="1277" height="157" alt="image" src="https://github.com/user-attachments/assets/fb2e1384-9667-43d7-9107-36afcea64f69" /> **Kuvassa projektikansion sisältö, jossa näkyvät README.md ja LICENSE -tiedostot.**

Seuraavaksi loin uuden tiedoston ja lisäsin sen versionhallintaan git add -komennolla. Tämän jälkeen tallensin muutokset git commit -komennolla ja lähetin ne GitHubiin git push -komennolla.

Git commit -komennolla tallennetaan muutokset paikalliseen versionhallintaan, ja git push siirtää nämä muutokset etävarastoon (GitHubiin).

Kuvasta myös näkee, että olen tehnyt ne jo aiemmin.**

<img width="957" height="383" alt="image" src="https://github.com/user-attachments/assets/a84036a3-59e1-4d1e-97b4-af4dd6ecd900" /> **Kuvassa GitHub-repository, johon uusi tiedosto on ilmestynyt git push -komennon jälkeen.**

Lopuksi tarkistin GitHubista, että uusi tiedosto oli siirtynyt sinne onnistuneesti. Tämä vahvisti, että versionhallinnan perusprosessi (add → commit → push) toimii oikein.

Tämä vaihe vastaa tehtävänannon vaatimusta, jossa luodaan repository, tehdään siihen muutoksia paikallisesti ja varmistetaan, että muutokset näkyvät GitHubissa.
