# H2 - Voileipä

## Tiivistelmät

### Sudo without password
- Sudo mahdollistaa komentojen ajamisen root-oikeuksilla ilman että kirjaudutaan root-käyttäjäksi.
- Salasanaton sudo (NOPASSWD) helpottaa automaatiota, koska komentoja voidaan suorittaa ilman käyttäjän syötettä.
- Tämä tehdään muokkaamalla sudoers-tiedostoa tai lisäämällä tiedosto hakemistoon /etc/sudoers.d/.
- Esimerkiksi ryhmälle voidaan antaa oikeudet: %group ALL=(ALL) NOPASSWD: ALL.
- On tärkeää olla varovainen, koska väärin määritetty sudo voi olla tietoturvariski.

**Oma huomio:** Salasanaton sudo on käytännössä pakollinen Ansiblelle, koska muuten automaatio pysähtyisi salasanaa kysyttäessä.

### XKCD Sandwitch
- Sarjakuva kuvaa tilannetta, jossa käyttäjä ei saa tehdä jotain ilman oikeuksia.
- Kun käyttäjä sanoo olevansa "administrator", hän saa oikeudet tehdä asian.
- Tämä viittaa siihen, että järjestelmissä oikeudet ratkaisevat mitä voi tehdä.
- Humoristinen tapa kuvata käyttöoikeuksien merkitystä tietojärjestelmissä.

**Oma huomio:** Sarjakuva liittyy sudoon, koska sudo antaa käyttäjälle tarvittavat oikeudet suorittaa komentoja järjestelmässä.

### Passwordless sudo with Ansible
- Ansible tarvitsee usein sudo-oikeudet suorittaakseen järjestelmänhallintatehtäviä.
- Salasanaton sudo mahdollistaa automaation ilman käyttäjän väliintuloa.
- Ansiblella voidaan määrittää sudo-oikeudet esimerkiksi copy-moduulilla luomalla tiedosto /etc/sudoers.d/.
- SSH-avaimet mahdollistavat kirjautumisen ilman salasanaa, mikä yhdistettynä sudoon tekee automaatiosta sujuvaa.
- Näin voidaan hallita useita koneita keskitetysti ilman manuaalista työtä.

**Oma huomio:** SSH + passwordless sudo yhdessä ovat Ansible-automaatioiden perusta.
