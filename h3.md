# H3 - Demoni

## Tiivistelmät

### Apache installed with Ansible - quick notes
- Apache voidaan asentaa Ansiblella käyttäen package-file-service -mallia.
- Ensin asennetaan paketti (apt), sitten kopioidaan konfiguraatiotiedostot (copy) ja lopuksi hallitaan palvelua (service).
- Konfiguraatiomuutokset eivät astu voimaan ennen kuin palvelu käynnistetään uudelleen.
- Ansible roolit jakavat kokonaisuuden selkeisiin osiin: tasks, files ja handlers.
- Handlerit mahdollistavat palvelun uudelleenkäynnistyksen vain tarvittaessa (esim. kun config muuttuu).

**Oma huomio: Tämä tekee automaatiosta tehokasta, koska turhia restartteja ei tehdä.**

### Handlers – running operations on change
- Handler on tehtävä, joka suoritetaan vain jos jokin muu task muuttuu ("notify").
- Tyypillinen käyttö on palvelun restart, kun konfiguraatiot muuttuvat.
- Handlerit määritellään erikseen ja niitä kutsutaan notify-avainsanalla.
- Tämä vähentää turhaa suorittamista ja tekee playbookeista tehokkaampia.
- Handlerit ajetaan vasta playbookin lopussa.

**Oma huomio: Handlerit on tärkeä osa “oikeaa” Ansiblea, ei pelkkä perus task-lista.**

### ansible-doc service
- service-moduulia käytetään palveluiden hallintaan (daemonit).
- Tärkeimmät parametrit:
   - name: palvelun nimi (esim. apache2)
   - state: present, started, stopped, restarted
   - enabled: käynnistyykö bootissa
- Esimerkki:
  - name: Start apache
    service:
      - name: apache2
      - state: started

## a.) Apassi

Asensin Apavhe2-webpalvelimen käsin käyttäen apt-paketinhallintaa. Käynnistin palvelun komennolla 'systemctl start apache2' ja varmistin toiminnan avaamalla selaimessa osoitteen http://localhost, jossa näkyi Apachen oletussivu.

Oletuksena verkkosivun tiedosto '/var/www/html/index.html' on root-käyttäjän omistuksessa, eikä tavallinen käyttäjä voi muokata sitä ilman sudo-oikeuksia. Muutin sekä tiedoston että hakemiston omistajaksi oman käyttäjäni komennolla 'chown', jotta tiedostoa voidaan muokata ilman sudoa. Pelkkä tiedoston omistajuuden muuttaminen ei riittänyt, vaan myös hakemiston oikeudet piti muuttaa, jotta tiedostoa voitiin muokata ilman sudo-oikeuksia.

Muokkasin index.html-tiedostoa ja varmistin selaimessa, että muutokset näkyvät oikein. Lopuksi tarkistin tiedoston oikeudet komennolla 'ls -l', joka vahvisti että omistaja on vaihdettu onnistuneesti.
